# Metadata & index formats

The launcher config is **two layers**:

1. **`GAME.TXT`** — human-edited metadata next to each game (source of truth).
2. **`GAMES.LST`** — machine index generated by `tools/scan-games.py` (what `BROWSER.COM` reads).

Do not hand-maintain `GAMES.LST` unless you know you need to; re-run the scanner after any catalog change.

---

## Per-game `GAME.TXT` (source of truth)

Place next to the game files, e.g. `GAMES\JILLOFTH\GAME.TXT`:

```ini
title=Jill of the Jungle
year=1992
genre=Platform
publisher=Epic MegaGames
exe=JILL.EXE
setup=
note=Epic MegaGames shareware platformer
```

| Field | Required | Notes |
|-------|----------|--------|
| `title` | yes* | Display name (*defaulted from folder if missing) |
| `year` | no | `1980`–`2099` |
| `genre` | no | Single primary genre (Action, Platform, …) |
| `publisher` | no | |
| `exe` | yes* | Launch file name in the game’s working directory |
| `setup` | no | Optional setup/config program (not used by v1 browser UI) |
| `note` | no | Shown in detail pane (one line; ~40 chars stored in index) |

Lines starting with `#` are comments. Keys are case-insensitive.

### Autogeneration

```bash
# After adding or changing games:
python tools/scan-games.py

# Prefer catalog titles/years when GAME.TXT is incomplete:
python tools/scan-games.py --apply-catalog

# Sort options: genre (default), year, title
python tools/scan-games.py --sort year
python tools/scan-games.py --no-headers
```

If `exe=` is missing, the scanner picks a launch file from `.EXE` / `.COM` / `.BAT`, preferring names like `start.bat`, `jill.exe`, `keen1.exe`, and skipping `setup.exe`, `cwsdpmi.exe`, `catalog.exe`, etc.

## Generated `GAMES.LST` (browser index)

Built by `tools/scan-games.py`. Written with **CRLF** and ASCII. The ASM browser **only** reads this file (from the current directory, or `C:\GAMES.LST`).

When the launcher is installed in a dedicated directory such as `C:\DGB\`, the browser reads `C:\DGB\GAMES.LST` and launches games from `C:\DGB\GAMES\<dir>\`.

```text
# comment
H|Action
G|BOUNCYBA|BOUNCYB.COM|Bouncy Ball|1983|Action|Public Domain|Short note
G|SNAKEMIN|SNAKEMI.COM|Snake Mini|1990|Action|Public Domain|
H|Platform
G|JILLOFTH|JILL.EXE|Jill of the Jungle|1992|Platform|Epic MegaGames|
```

| Kind | Fields |
|------|--------|
| `H\|title` | Section header (not launchable) |
| `G\|dir\|exe\|title\|year\|genre\|publisher\|note` | Game |

- `dir` is relative to the launcher directory’s `GAMES\` folder (may include subdirs, e.g. `COMMANDE\KEEN`)
- At launch: `CHDIR` to `<launcher>\GAMES\<dir>`, then `EXEC` `<exe>`
- ASCII only; keep total line length ≤ 120 characters
- Limits in v1: up to **64** entries (including headers/spacers); title ≤ 32 chars stored

## Sample catalog (`tools/sample-catalog.json`)

Used by:

- `tools/fetch-samples.py` — download free/shareware packs into `booth/GAMES/`
- `tools/scan-games.py --apply-catalog` — fill missing metadata fields

Extend the JSON when you want a reproducible demo set without committing binaries.

### AI-assisted metadata cleanup

After scanning a mounted DOS image, you can use an AI agent to help fill in missing `GAME.TXT` fields. Keep the output conservative and review it before rescanning.

Prompt template:

```text
You are helping prepare a DOS game launcher catalog.

I have a folder tree under booth/GAMES/ with DOS game binaries and per-game GAME.TXT files generated by a scanner.

Task:
- Inspect the discovered binaries and existing GAME.TXT files.
- Infer the most likely title, year, genre, publisher, executable, setup program, and one-line note.
- Keep entries accurate and conservative; do not invent facts when the binary name is unclear.
- Prefer the real launch executable over setup or catalog tools.
- Preserve DOS 8.3-friendly filenames and backslashes in paths.
- Output only the updated GAME.TXT contents, one file at a time.

If information cannot be determined with confidence, leave the field blank and say why.
```
